This guide walks you through creating a working counter component that increments in real time over WebSocket, using the same pattern as the JReactive demo application.

<Info>
  **Prerequisites**: Java 21, Maven 3.8+, and a Spring Boot 3.2 project skeleton. You can generate one at [start.spring.io](https://start.spring.io) — select **Spring Web** as the only dependency.
</Info>

<Steps>

<Step title="Add the JReactive starter dependency">

Open your project's `pom.xml` and add the JitPack repository and the Spring Boot starter:

```xml pom.xml
<repositories>
  <repository>
    <id>jitpack.io</id>
    <url>https://jitpack.io</url>
  </repository>
</repositories>

<dependencies>
  <dependency>
    <groupId>com.ciro</groupId>
    <artifactId>jreactive-starter-spring</artifactId>
    <version>0.0.1-SNAPSHOT</version>
  </dependency>
</dependencies>
```

The starter pulls in `jreactive-core`, `jreactive-runtime-jvm`, and `jreactive-runtime-js` automatically, along with Spring Boot Web, WebSocket, and Validation starters.

<Note>
  You also need to configure the annotation processor (`jreactive-apt`) in your Maven Compiler Plugin. See the [Installation](/installation) page for the full compiler plugin configuration.
</Note>

</Step>

<Step title="Create the base component">

Create a base class that holds your state and declares the `@Call` methods. Keeping logic in an abstract base class lets you reuse it across multiple component variants.

```java BaseCounter.java
package com.example.myapp;

import com.ciro.jreactive.HtmlComponent;
import com.ciro.jreactive.State;
import com.ciro.jreactive.annotations.Call;

public abstract class BaseCounter extends HtmlComponent {

    @State
    public int count = 0;

    @Call
    public void increment() {
        count++;
    }

    @Call
    public void reset() {
        count = 0;
    }
}
```

`@State` marks `count` as reactive state. JReactive tracks it and pushes a DOM diff to the browser whenever its value changes after a `@Call` method returns.

`@Call` marks a method as browser-invocable. JReactive makes it available to template event handlers. By default, `@Call` methods run synchronously (`sync = true`) and trigger a state sync after they complete.

</Step>

<Step title="Create the leaf component with a template">

Create a concrete component that extends `BaseCounter` and provides the HTML template:

```java CounterLeaf.java
package com.example.myapp;

import com.ciro.jreactive.annotations.Client;
import com.ciro.jreactive.annotations.StatefulRam;

@Client
@StatefulRam
public class CounterLeaf extends BaseCounter {

    @Override
    protected String template() {
        return """
            <div style="border: 2px dashed #666; padding: 10px; margin: 10px;">
                <h3>Counter</h3>
                <p>Count: <strong>{{count}}</strong></p>

                <button @click="increment()">+1</button>
                <button @click="reset()">Reset</button>
            </div>
        """;
    }
}
```

**Template syntax:**
- `{{count}}` — interpolates the value of the `@State` field named `count`
- `@click="increment()"` — binds the button's click event to the `@Call` method `increment()`

**Annotations:**
- `@Client` — this component renders on the client side after the initial page load (CSR mode), keeping server load low for interactive leaf nodes
- `@StatefulRam` — state is stored in-process memory (Caffeine cache); suitable for development and single-instance deployments

</Step>

<Step title="Create the page and register a route">

Create a page component that acts as the route root. It embeds `CounterLeaf` as a child component using its class name as an HTML tag:

```java HomePage.java
package com.example.myapp;

import com.ciro.jreactive.HtmlComponent;
import com.ciro.jreactive.router.Route;

@Route(path = "/")
public class HomePage extends HtmlComponent {

    @Override
    protected String template() {
        return """
            <div class="page">
                <h1>My First JReactive App</h1>
                <CounterLeaf />
            </div>
        """;
    }
}
```

`@Route(path = "/")` maps this component to the root URL. JReactive's `RouteProvider` picks it up automatically when scanning the Spring context.

Child components are referenced by their simple class name as self-closing tags: `<CounterLeaf />`.

</Step>

<Step title="Create the Spring Boot application entry point">

If you don't already have one, create the standard Spring Boot application class:

```java Application.java
package com.example.myapp;

import org.springframework.boot.SpringApplication;
import org.springframework.boot.autoconfigure.SpringBootApplication;

@SpringBootApplication
public class Application {

    public static void main(String[] args) {
        SpringApplication.run(Application.class, args);
    }
}
```

JReactive's `JReactiveAutoConfiguration` registers itself automatically via Spring Boot's auto-configuration mechanism. You do not need to import any JReactive configuration class manually.

</Step>

<Step title="Configure the application">

Create `src/main/resources/application.yml` with the minimal JReactive configuration:

```yaml application.yml
server:
  port: 8080

jreactive:
  store:
    type: ram
  ws:
    persistent-state: true
    enabled-backpressure: true
    max-queue: 1024
    flush-interval-ms: 16
```

`store.type: ram` uses an in-process Caffeine cache for session state. See [Installation](/installation) for the full list of configuration properties including the Redis hybrid store.

</Step>

<Step title="Run the app">

Build and run your project:

```bash
mvn spring-boot:run
```

Open [http://localhost:8080](http://localhost:8080) in your browser. You should see your `HomePage` rendered with the embedded `CounterLeaf`. Click the **+1** button — the counter updates in real time without a page reload.

<Tip>
  Open the browser's DevTools Network panel and filter by **WS**. You can watch the WebSocket frames carrying state diffs as you click the buttons.
</Tip>

</Step>

</Steps>

## What just happened?

When you clicked **+1**:

1. The browser runtime sent a `call` message over the WebSocket identifying the component ID and method name `increment`.
2. JReactive located the `CounterLeaf` instance for your session, acquired its lock, and invoked `increment()`.
3. `count` changed from `0` to `1`. JReactive detected the delta, re-rendered the component, and computed the HTML diff.
4. The diff was sent back over the WebSocket as a patch message.
5. The browser runtime applied the patch to the DOM — only the `<strong>` element containing the count was updated.
